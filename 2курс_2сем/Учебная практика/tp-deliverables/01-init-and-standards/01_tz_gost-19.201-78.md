## 3. НАЗНАЧЕНИЕ РАЗРАБОТКИ

### 3.1 Функциональное назначение
Программа предназначена для:
- визуализации силовых линий заданного векторного поля a;
- визуализации эквипотенциальных поверхностей заданного векторного поля b;
- численного расчёта потока P поля a через поверхность S;
- численного расчёта циркуляции C поля a по контуру поверхности S;
- нахождения потенциала U поля b и работы W.

### 3.2 Эксплуатационное назначение
Программа предназначена для запуска на персональных компьютерах под управлением ОС Linux или Windows. Результатом работы программы является окно с трёхмерной визуализацией и консоль с численными результатами.

## 4. ТРЕБОВАНИЯ К ПРОГРАММЕ

### 4.1 Требования к функциональным характеристикам

#### 4.1.1 Состав выполняемых функций
Программа должна обеспечивать возможность выполнения следующих функций:

1. **Расчёт класса поля a**:
   - вычисление дивергенции div a;
   - вычисление ротора rot a;
   - классификация поля (соленоидальное, потенциальное, гармоническое).

2. **Расчёт потока P поля a через поверхность S**:
   - методом Гаусса-Остроградского (с замыканием координатными плоскостями);
   - прямым вычислением поверхностного интеграла.

3. **Расчёт циркуляции C поля a по контуру поверхности S**:
   - методом Стокса;
   - прямым вычислением криволинейного интеграла.

4. **Исследование поля b**:
   - проверка потенциальности (rot b = 0);
   - вычисление потенциала U;
   - проверка потенциала;
   - вычисление работы W от O(0,0,0) до M(1,1,1).

5. **Визуализация**:
   - отображение поверхности S (единичная сфера в первом октанте);
   - отображение силовых линий поля a;
   - отображение эквипотенциальных поверхностей поля b;
   - отображение координатных осей, сетки, подписей;
   - возможность вращения и масштабирования 3D-сцены.

#### 4.1.2 Организация входных и выходных данных

**Входные данные:**
- векторное поле a = (y², z², x²);
- векторное поле b = (2xy³z + y, 3x²y²z + x, x²y³ + 4z³);
- поверхность S: x² + y² + z² = 1, x ≥ 0, y ≥ 0, z ≥ 0.

**Выходные данные:**
- численные значения P, C, U, W;
- 3D-визуализация силовых линий и эквипотенциальных поверхностей;
- текстовый протокол расчётов.

#### 4.1.3 Временные характеристики
Время выполнения расчётов — не более 5 секунд.  
Время генерации 3D-визуализации — не более 10 секунд.

### 4.2 Требования к надёжности

#### 4.2.1 Обеспечение надёжного функционирования
Программа должна корректно обрабатывать:
- некорректные входные данные (выход за границы, нечисловые значения);
- деление на ноль при численных расчётах;
- ошибки открытия/чтения файлов (при наличии файлового ввода).

#### 4.2.2 Восстановление после сбоев
При возникновении ошибки программа должна выводить диагностическое сообщение на русском языке и завершать работу с кодом возврата 1.

#### 4.3.1 Климатические условия

### Приложение А — Вариант 5. Поля и поверхность

**Поверхность S:** часть единичной сферы x² + y² + z² = 1, лежащая в первом октанте (x ≥ 0, y ≥ 0, z ≥ 0).

**Поле a:**  
a = y²·i + z²·j + x²·k

**Поле b:**  
b = (2xy³z + y)·i + (3x²y²z + x)·j + (x²y³ + 4z³)·k
