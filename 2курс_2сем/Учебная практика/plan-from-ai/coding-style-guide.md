## 2. ИМЕНОВАНИЕ

| Что | Стиль | Примеры |
|-----|-------|---------|
| Переменные | **snake_case** | `start`, `interval_count`, `max_value`, `root_secant`, `iter_secant` |
| Массивы | ALL_CAPS | `AVec`, `IVECT`, `NEWVEC` |
| Константы | ALL_CAPS | `SIZE` |
| Функции | **camelCase** | `readMatrix`, `printMatrix`, `sortColumnsBySum`, `columnSum` |
| Макросы | snake_case или одна буква | `step_lenth`, `F(x)`, `G(x)` |

---

## 3. ОТСТУПЫ

- **2 пробела** — стандарт для 1-го семестра (дефолт)
- **4 пробела** — для крупных/многофункциональных программ
- **Табы — НЕ ИСПОЛЬЗУЮТСЯ**

Рекомендация: для текущей практики ставь **2 пробела**.

---

## 4. СКОБКИ

K&R (египетский стиль) — везде:

```cpp
int main() {
  if (condition) {
    for (int i = 0; i < n; ++i) {
      // тело
    }
  }
}
```

---

## 5. КОММЕНТАРИИ

- Язык: **русский** (для 1-го курса), английский — редко
- Каждая переменная — inline `//` после объявления
- Каждый логический блок — `/* комментарий */` перед ним
- Разделители секций — блоки в рамках:

```
/****************
* Основное тело *
****************/
```

---

## 6. СТРУКТУРА ФАЙЛА

```
1. Шапка (бокс с кафедрой)
2. #include <iostream>, <cmath>, <iomanip>, <fstream>, <string>
3. using namespace std;
4. #define ... (если есть)
5. Глобальные переменные (если есть)
6. Прототипы функций (если много функций)
7. int main() { ... exit(0); }
8. Реализация функций (после main)
```

Любимые разделы внутри main:
- `Подключение препроцессора и библиотек`
- `Переменные`
- `Объявление переменных`
- `Входной контроль`
- `Основное тело`
- `Вывод полученной информации`

---

## 7. ВАЛИДАЦИЯ ВВОДА

Обязательная проверка каждого ввода с сообщением об ошибке на русском:

```cpp
if (tol > 1) {
  cout << "Ошибка: значение tol не может превышать 1." << endl;
  exit(1);
}
```

Паттерн: проверка → сообщение → `exit(1)`.

## 8. ШАБЛОН КОДА

```cpp
/****************
* Основное тело *
****************/
int main() {
  // Ввод исходных данных
  cout << "Введите A: ";
  cin >> a;

  // Проверка ввода
  if (a <= 0) {
    cout << "Ошибка: A должно быть больше 0." << endl;
    exit(1);
  }

  // Основные вычисления
  for (int i = 0; i < n; ++i) {
    x = a + i * h;             // Приращение аргумента
    y = (2 - cos(x));          // Значение функции
    cout << "x = " << x << ", F(x) = " << y << endl;
  }

  exit(0);
}
```

## 9. ИДИОМЫ

| Паттерн | Где |
|----------|-----|
| `using namespace std;` | Всегда |
| `setlocale(LC_ALL, "Russian");` | Для русского вывода в консоль |
| `cout << fixed << setprecision(N);` | Вывод с N знаков после запятой |
| `exit(0)` / `exit(1)` | Ручное завершение |
| `pow(x, 3)` | Степень |
| `fabs()` | Модуль для float |
| `++i` (префиксный инкремент) | В циклах |
| `new int*[rows]` + цикл `new int[cols]` | Двумерные массивы |
| `delete[] matrix[i]`; `delete[] matrix` | Очистка 2D массива |
| `sizeof(AVec)/sizeof(AVec[0])` | Длина массива |

---

## 10. ПРАВИЛО "20 ОПЕРАТОРОВ"

Стандарт структурного программирования: **≤20 операторов на модуль** (лучше 7-10).
В твоих старых работах:
- Типичный модуль = **50-110 строк**, ~10-20 операторов
- Тело циклов редко превышает 15 операторов
- Разбивка на секции помогает держать лимит
